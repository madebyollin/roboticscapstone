Blog Post #8 Part 1

iOS Frontend Progress Report

This week, we made good progress on the iOS app. We also completed a server that can communicate with both the robot and instances of the iOS app. This allows the iOS app to control the robot directly.

Currently, there are a few functions that are completed, namely: asking the robot to speak, and telling the robot to go to a specific room. In addition, the app implements a "pairing" abstraction that ensures that only one person can control the robot at a time. Lastly, the app is very resilient to issues with the server or robot going down; both the server and the app can run for days even if the other one goes down.

The remaining work is to test navigation on the real robot (once the map of the hallway is done), and add some stretch goal features such as a left handed mode and speaking directions/route info.
